# AR宝箱解谜项目 README\.md

# 一、项目简介

本项目为**Unity EasyAR 图片识别 AR 实景解谜案例**，基于 URP 管线开发。实现图片识别生成宝箱、4位密码锁交互、开箱动画、钥匙升起、金色粒子特效、材质动态变色、xLua 热更新事件回调等完整演示流程，整体风格统一、动画流畅，满足 AR 实景交互精品演示标准。

# 二、运行环境版本

- **Unity**：2022.3.51f1c1

- **EasyAR Sense**：4002.0.0+4956.1ec38c1ad

- **URP**：14.0.11

- **UniGLTF**：1.27

- **DOTween**：v1.2.825

- **xLua**: v2.1.15
  

# 三、运行配置说明

## 1\. EasyAR 配置

- 在 EasyAR 官网注册应用，获取 Android 平台 License Key

- 将 Key 填入场景中 EasyARController 的 License 字段

- 配置识别图，绑定 ImageTarget 识别对象

- AR 相机切换为 URP 渲染，避免管线报错
  

## 2\. xLua 配置

- PlayerSettings 宏定义添加：`XLUA;HOTFIX_ENABLE`

- 顶部菜单执行：XLua → Generate Code 生成绑定代码

- Lua 脚本放置在 Assets/Lua 目录，打包可正常读取

## 3\. Android 打包配置

- 切换 Android 平台，仅勾选 ARM64\-v8a

- 使用 Unity 内置 JDK，配置对应版本 NDK

- 打包使用 Build 模式（无需 ADB 设备连接即可导出完整 APK）

## 4. 运行流程
- 打开主场景：`Assets/Scenes/SampleScene`
- 设备对准AR识别图，识别成功后场景自动生成宝箱模型
- 点击宝箱交互区域，弹出4位齿轮密码锁
- 拨动密码轮输入正确密码，触发完整开箱动画、粒子特效、宝箱材质发光变色
- 动画播放完毕，自动执行Lua脚本开箱回调并打印运行日志
# 四、场景实现思路

## 1\. 模型资源处理与模型加载方案

初期计划使用 **GLTFast** 插件加载外部 GLB 模型，但存在严重版本兼容问题：当前 Unity 版本无法正常导入模型、贴图丢失、网格解析失败，降级多个旧版本依旧报错。

最终更换为 **UniGLTF** 插件，成功解析 GLB 宝箱与钥匙模型。

模型前期在 **Blender** 中手动拆分重构：

- 将宝箱拆分为：底座、锁扣、箱盖三个独立网格

- 单独修改箱盖、锁扣的旋转轴心，适配开盖、解锁动画

- 4个密码拨轮单独拆分独立模型，保证每轮可单独旋转控制

## 2\. 密码锁交互实现
采用**物理模型旋转 \+ 数值映射**的方式实现 4 位密码锁：

- 四个密码滚轮为独立模型，绑定鼠标点击滑动旋转逻辑

- 根据滚轮的**局部旋转角度区间**映射对应 0\-9 数字

- 每拨动一次自动采集数字，凑齐4位自动校验密码

- 密码正确进入开箱序列，错误清空输入并触发 Lua 错误回调

完全脱离UGUI纯数字按钮，使用**3D模型物理拨动交互**

## 3\. 动画系统设计

全程使用 DOTween 序列管理动画时序，采用「并行\+串行」组合编排：

- 锁扣旋转解锁动画

- 箱盖平滑开盖动画（OutSine 缓动，手感柔和）

- 钥匙从宝箱内部缓缓升起

- 材质亮度渐变同步动画节奏

## 4\. 材质发光开发尝试

项目使用 UniGLTF 自带的 **UniUnlit 无光照Shader**，该Shader**无 Emission 自发光通道、不支持 HDR**，无法实现标准发光 bloom 效果。

为了不破坏 GLTF 模型材质适配性、不改动源码Shader，尝试方案：

- 通过代码动态放大主纹理 \_Color 的 RGB 数值（突破1\.0实现超亮）

- 使用 `Mathf.Clamp01` 将 0\~6 的动画强度归一化为 0\~1 插值系数

- 实现从原色到高亮色的线性渐变

最终因无 HDR 与 Bloom 后处理加持，**高亮发光效果不明显，未达到理想发光质感**，属于本项目主要未完成优化点。

## 5\. 粒子特效系统

配置金色光点粒子特效，实现开箱瞬间喷发效果；粒子挂载跟随脚本，锁定XY与旋转，仅跟随钥匙Z轴做微量偏移，贴合开箱特效位置。因初次调试粒子参数不熟练，粒子疏密、轨迹、生命周期未做到极致精致。

## 6\. xLua 热更新预留模块

按照需求预留 Lua 业务事件层：

- 开箱动画完全结束后主动调用 Lua 开箱成功回调

# 五、项目亮点

- **模型精细化处理**：全程使用 Blender 对 GLB 模型二次拆分、调整轴心、分离部件，实现箱盖、锁扣、密码轮独立可控，是纯代码动画能流畅运行的基础。

- **真实3D密码交互**：不使用UGUI数字按钮，通过模型旋转角度映射数字，实现拟真物理拨动密码锁体验，交互质感远超普通UI输入。

- **完整解耦架构**：AR识别、模型动画、特效、材质、Lua事件分层清晰，代码结构规范、可维护。

- **严格适配AR管线**：全程基于URP开发，适配移动端AR渲染标准。

- **满足所有硬性考核点**：EasyAR识别、GLB模型加载、DOTween动画、粒子特效、材质变色、xLua事件回调全部完成。

# 六、待优化与不足

- **材质发光效果不足**：受限于 UniUnlit 无自发光、无HDR，仅靠主色提亮无法实现真实光晕发光，后期可替换 Lit 材质 \+ Bloom 后处理优化质感。

- **粒子特效细节不足**：初次调试粒子参数熟练度不足，粒子轨迹、速率、渐变、收尾消散效果未打磨到精致级别。

- **场景打光缺失**：项目全程未做精细实时光照、补光、阴影，模型质感偏平。

- **UI界面简陋**：仅实现基础提示文字，无界面美化、弹窗、状态提示界面。

- **音效系统未完善**：仅加入两个音效，丰富度不足。

# 七、总结

本次项目完整实现了**AR图片识别、3D密码解谜、全套开箱动画、粒子特效、材质渐变、Lua热更新事件预留**所有考核需求。开发过程中解决了模型导入兼容、GLB模型拆分适配、AR渲染管线适配、Lua交互对接等多项问题。因开发时间有限，视觉渲染、特效细节、UI体验仍有较大优化空间。
